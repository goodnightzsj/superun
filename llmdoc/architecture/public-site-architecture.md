## Content model
- Template feed and template catalog data now live in the generated JSON inputs `src/data/template-directory.json` and `src/data/template-index.json`. `src/data/template-directory.ts` derives `previewSlug` from `visitUrl`, exports the route-slug lookup used by `/home` and `/web/community-view/[slug]`, and parses long-form `websiteIntroduction` content into renderable narrative blocks alongside the merged home/template catalog records. The first 40 home feed records are periodically re-synced from the upstream `AgentCommunity/queryByCategory` response, including `previewImage`, `deviceType`, and localized category mapping, while `TemplateCard` still retains a placeholder shell fallback for future missing-image states instead of emitting broken `img` tags (`src/data/template-directory.ts:1`, `src/data/template-directory.ts:36`, `src/data/template-directory.ts:50`, `src/data/template-directory.ts:57`, `src/data/template-directory.ts:75`, `src/data/template-directory.ts:89`, `src/data/template-directory.ts:112`, `src/components/TemplateCard.astro:12`, `src/components/TemplateCard.astro:33`, `src/styles/global.css:503`)
- Docs records plus shared footer/navigation metadata remain in `src/data/site.ts`
- Public locale/copy state, placeholder text sequences, and shared community copy live in `src/data/fidelity.ts`
- `/docs/[slug]` prerenders directly from `src/data/site.ts`; there is no separate active `astro:content` docs source
